# FAQ

## What is Eclipse?

Eclipse is a private autonomous AI operator that studies public market data, wallet behavior, social context, and narrative motion. It uses that context to generate structured alerts, paper-trade simulations, outcome learning, and operator-controlled social actions.

## Is Eclipse open source?

No. This repository is a public technical overview. The private runtime and backend source code are not published.

## Why have a public repository if the runtime is private?

The repository gives the project a verifiable public identity and explains the system at a high level without leaking code, credentials, wallet data, prompts, private memory, or strategy implementation.

## Does this repository contain trading logic?

No. It describes system areas and security boundaries. It does not include executable strategy code, private thresholds, wallet lists, transaction signing logic, or backend implementation details.

## Does Eclipse control real funds?

This repository only describes the system publicly and references paper-trade simulation. It does not publish private wallet material, transaction infrastructure, signing logic, or operational trading configuration.

## What does paper-trade simulation mean here?

Paper trading means the system can record simulated entries and exits, including entry market cap, simulated size, exit reason, and PnL. It is used for research and feedback. It is not proof of future live performance.

## Is this financial advice?

No. Eclipse is experimental software. Nothing in this repository is financial advice, trading advice, or a recommendation to buy or sell any asset.

## What makes Eclipse technically different?

The public version can safely say this: Eclipse combines wallet reputation, market structure, narrative context, risk evidence, simulation, and memory in one agent loop. It is designed to learn from outcomes rather than treat any single signal as absolute.

## How are Hermes and OpenClaw used?

Hermes is used as a local agent compatibility and continuity lane around Eclipse. OpenClaw is used as an optional local sidecar/gateway for agent workspace coordination and heartbeat-style checks. Neither tool makes the private Eclipse runtime public, and neither publishes private prompts, wallet data, credentials, raw logs, or strategy implementation.

## Can contributors submit code?

No private runtime code is accepted here. Issues may be used for public documentation feedback or non-sensitive questions only.

## What should never be posted in an issue?

Never post API keys, wallet data, private logs, screenshots with account identifiers, runtime state, exploit details, credentials, private prompts, or anything that could expose operational infrastructure.
